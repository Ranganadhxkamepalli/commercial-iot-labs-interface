Read the Objectives

The Intel® IoT Developer Hub is a web-based service running on the gateway that allows you to have a hands-on sensor-to-cloud experience in a very short time period. It has built-in tutorials to teach you about tools like the visual programming interface called Node-RED* and the Wind River Helix App Cloud, including sample code to develop the app in the cloud. The Intel® IoT Developer Hub provides ways to easily manage the administrative settings of the gateway. The Intel® IoT Developer Hub lets you, as the developer, add packages and develop your apps. When development is complete, you can save a hardened OS image onto a USB flash drive.

By the end of this module, you should be able to:

  • Connect gateway with Helix App Cloud
  • Create graphical view of Sensor data
  • Access repositories and manage packages on gateway
  • Access Gateway Administration tools and resources
  • Getting Familiar with Cloud Commander

Introduction to Intel Iot Dev Hub

Introduction to Intel Iot Dev Hub

  1. Open your browser and go to IP address of your gateway to access IoT Developer Hub.
  2. Login to the Intel® IoT Gateway Developer Hub using root for both the username and the password.
  3. Use the Intel® IoT Gateway Developer Hub to explore and extend the basic capabilities of the Gateway.

Advantages of using Intel IoT Dev Hub

The advantages of using Intel IOT Developer Hub gives you,

  • Live Dashboard
    • Summary view of gateway information
    • Graphical view of sensor data
    • Notification of available OS updates
  • Plug and play sensors
    • Shows live data from connected sensors
    • Easy management of connected sensors via Node-RED
  • Simple access to repositories and management of popular package

    • Easily list installed packages

  • Clear and simple documentation
    • One location to find how-to tutorials and videos, and quick links to key online content

Package Management Overview

Package Management Overview

As periodic updates are released, it is important to ensure that you are keeping your gateway development environment up to date.

The Developer Hub interface provides an easy mechanism to perform these package updates.

List Packages

List Packages

The update indicator in the top right corner shows the number of updates needs to be installed.

Before clicking to update, it is important that you first update the Developer Hub itself.

This will ensure that the Developer Hub has the latest code to deal with the rest of the updates since it is performing the updates.

Update Repositories

Update Repositories

Click on the Add Repo button. It will ensure we that all latest packages to be updated. Intel_Repository will be located by default.

Click on the Update Repositories button.

After a minute or so, the button text will change to Updated.

Now close the window and there will be updated list of packages under the packages tab.

Update Developer Hub Package

Update Developer Hub Package

Now click on iot-developer-hub package from the list and a new screen will allow you to update or uninstall that package.

Proceed to update the iot-developer-hub package.

Click on Update button

As the dialog box indicates, this will take a few minutes. It has to download the package update from the internet and install it (which restarts the Developer Hub Interface).

This all takes some time, but once it is finished, you can refresh the browser tab pointing to the gateway IP address and see the Developer Hub up and running again.

If required, log in again to developer hub and click on Packages tab.

Package Management Overview

Package Management Overview

This is optional step as updating all the packages will be time-consuming and also depends on the bandwidth available.

It is ready to update the rest of the packages. To do this, click on the Install Updates button under the packages tab.

It will update the rest of the packages in one action.

Creating Deployable Gateway Image

Creating Deployable Gateway Image

Create a deployable OS Image for production use based on the OS and software running on this gateway.

The deployment OS image includes all applications, data files, and OS updates from the running gateway, but changes can be made to the OS settings and some configuration files to harden the OS to provide better security in a deployment environment.

The OS image is hardened and saved to a USB drive while leaving the development gateway OS unchanged.

The replicated image can quickly be deployed onto another system for evaluation & testing. This enables validation of an IOT system prior to field deployment.

LuCI - Network Configuration Interface

LuCI - Network Configuration Interface

LuCI is an open-source administrative tool for managing Linux system settings, network information, logs, processes, and more.

Use a Web browser to navigate to the Gateway IP address. For example, type https://Gateway_IP_Address in the URL box, where Gateway_IP_Address is the IP address for your Gateway in the format 123.456.1.1

Make sure to add https for the LuCI interface

The Status menu shows the Overview option that displays an operating system configuration summary and other information. Following functionalities available under Status Menu

  • Up-time since last boot
  • Memory usage
  • Network and wireless settings
  • Real-time graphs for performance Review
  • Log files

The System menu provides access to screens that manage common Gateway settings such as the hostname, system time, password, backup/restore, and more. Following functionalities available under System Menu

  • Change the Date and Time
  • Change the Name (Hostname)
  • Change the Root User Password
  • Manage Systemd Units (Services, Resources, etc.)

Gateways that use Intel® IoT Gateway Technology may use wired Ethernet and wireless connections, Bluetooths, and other network interfaces. Use the Network menu to manage and troubleshoot networking functions. Following functionalities available under Network Menu

  • Changing Wireless Network Settings
  • Change the Wireless SSID, Channel, and Visibility
  • Set up the Gateway as a Client or an Access Point
  • Change the Wireless Encryption Type
  • Changing the Wireless SSID Password (Key)
  • Change the LAN and WAN Interfaces for DHCP and Static Addressing

The interface was created to support the Unified Configuration Interface (UCI), which is covered in detail at Luci Documentation and UCI Wiki

Overview of Cloud Commander

Overview of Cloud Commander

Cloud Commander* is a web based file manager with console and editor. Using Cloud Commander, you can work with files, directories and programs via your browser from any computer, mobile or tablet. It's a quick and easy way to take an application you've developed locally, upload it to your gateway, and test, edit, iterate directly from the Intel® IoT Gateway Developer Hub on your IoT Gateway that uses Intel® IoT Gateway Technology.

Access Cloud Commander

Access Cloud Commander

You can access Cloud commander from the Intel IoT Developer hub.

Click on the Administration Icon and then click on Launch button under folder icon.

If dialog box opens up for authentication then enter root in username and password text box.

Get Started with Cloud Commander

Get Started with Cloud Commander

There are two sections showing the folder listing on your IoT gateway. From here you can navigate to any folder or file on your IoT gateway just by clicking on the folder name.

Here are the primary features of Cloud Commander.

  • F1 Opens a Help screen
  • F2 Renames a file
  • F3 Views a file
  • F4 Edits a file
  • F5 Copies a file. You will be prompted for the location to copy to. By default it will be whatever location you are in on the right side file / folder browser.
  • F6 Moves a file.You will be prompted for the location to copy to. By default it will be whatever location you are in on the right side file / folder browser.
  • F7 Create a new folder
  • F8 Delete a file (you will be prompted to confirm)
  • F9 Opens a menu of these options including extras such as Pack and Extract which uses Tar). You can also open this menu or a subset by right clicking on a file, folder, or blank space.
  • F10 Opens a settings menu for Cloud Commander
  • >_~ Opens a web based terminal to your IoT gateway. Very powerful way to quickly execute a command or commands directly on your IoT gateway from the web browser

To copy files to/from your local system, just drag them to/from the Cloud Commander interface. You can also right click in Cloud Commander and choose Upload or right click on a file in Cloud Command and choose download. By right clicking, you'll also see Upload to Cloud or Upload From Cloud. This is just like coping to/from your local system but supports directly accessing common cloud locations such as Dropbox, Google Drive, Box, and Amazon Cloud.
