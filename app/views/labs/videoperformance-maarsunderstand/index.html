Error while running a program due to NODE_PATH path not set

Error while running a program due to NODE_PATH path not set

  • If you see an error as shown in figure while running your XDK program, your NODE_PATH might not be set correctly

  • On your gateway terminal set the path variable as below

    • export NODE_PATH=/usr/lib/node_modules/

  • Run the program again from XDK and this error should not come up

    • Errors due to other instances of XDK or Node-Red running

    Errors due to other instances of XDK or Node-Red running

  • If you see an error as shown in figure while running your XDK program, make sure if you are running another instance of XDK to stop it

  • Also make sure to stop your Node-Red service in case it is running

  • Open a terminal to your Gateway and give following command:

    • systemctl stop node-red-experience

  • Check that service has stopped

    • systemctl status node-red-experience

  • Follow this same procedure if you also get error like "ERROR: 'firmata' platform required but not detected: 3 -> Intel DE3815"
