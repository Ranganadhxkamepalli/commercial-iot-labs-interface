Install and Run the Custom Administration Interface

You can monitor the status of your edge network system using NodeJS on the back end and HTML, CSS, and JavaScript on the front end. Write system condition based monitoring triggers, record system errors, and perform edge network analytics.

By the end of this module, you should be able to:

  • Deploy a web-based administrative interface on your Intel® IoT Gateway.
  • Have an understanding of how you could build your own web-based monitoring system.

Install the supplied administrative interface

  1. Download the admin-interface.zip.

  2. Transfer the archive file to your Intel® IoT Gateway.

  3. Use PuTTY to SSH into the Intel® IoT Gateway.

  4. Navigate to the directory with the zip file and unzip it.

    $ unzip admin-interface.zip

  5. After the zip file has been uncompressed, go into the resulting folder.

    cd admin-interface

  6. Run server.js as a Node process.

    node server.js

    This will start a NodeJS HTTP server that will provide the front end user interface.

  7. Visit "http://{Your Gateway IP address}:3000" in your development computer's web browser.

  8. It will show you in a browser a simple, sample web based interface that will allow you to create, edit and delete automatic actions (triggers)

    Before we explore more options let's populate this interface with some static data

  9. Populate this sample interface by running initialize-db.js script from the lab-solution-monitor-daemon repository downloaded in previous "Databases at the Edge" lab

    Open a new console to your Gateway and give the following commands:

    cd ~/lab-solution-monitor-daemon
cd initialize
    npm install
    node initialize-db.js

    Note that once it runs this initialization script requires you to exit it by pressing Ctrl-C.

    This will create an LCD actuator, a temperature sensor and a couple of basic automatic rules. Refresh your browser on your computer to see these on the admin interface web page

    Explore the interface for condition based monitoring triggers, record system errors.

    In the next lab, you will write most complex automatic rules and a rules engine to the run these rules on an edge network.
