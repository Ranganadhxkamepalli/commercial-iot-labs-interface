Overview of Labs: What are we doing and where are we going?

The labs in the Intel Commercial IoT Workshop are build in a progression which will result in each attendee building their own automated and secured, multi-device IoT network.

The set of labs start with a basic introduction to the Intel IoT Gateway. From there, we will begin to build a complete and deployable IoT architecture based on Intel technology.

The next steps will include experimenting with sensors and actuators from the Grove IoT Development Kit, setting up a database that will store sensor data on the edge network, install a customized user interface that administrators can use to monitor edge network activity, integrate with the Helix Device Cloud for enterprise manageablity and administration. We will then use Intel Security tools to lock down the gateway and ready it for deployment. Finally, we look at integration with 3rd party cloud providers such as Microsoft Azure, GE Predix, Amazon AWS and IBM BlueMix.

  1. Open your browser and go to IP address noted down in last step to access IoT Developer Hub.

Download Putty

  1. Windows does not come with a built-in terminal emulator so download and use the PuTTY client.
  2. Visit the PuTTY download page.
  3. Under the "For Windows on Intel x86" heading, click on the "putty.exe" link to download the latest release version to your computer.

Launch Putty and type in Your IP Address to connect

  1. Double-click putty.exe on your computer to launch PuTTY.
  2. Enter IP address of the Gateway
  3. You can login with the username root and the password root.

Open a Console and use the ssh command to connect

  1. Open Terminal
  2. Type $ ssh root@<>. Replace <> to IP address of your gateway. E.g. root@192.168.3.3
  3. Enter root as password

Change the password of your Intel IoT Gateway

At the console type the follow:

passwd

Follow the prompts at change your password.
